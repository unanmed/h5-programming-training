# 第一章 第二节 判断与循环

在本节中，你将学会在js非常基础且常用的语句——判断语句与循环语句

## 一 判断语句

判断语句是一个很常见的语句，主要形式有以下三种：

```js
// 形式一 以下condition指成立条件，code指条件成立时执行的代码
if (condition) { code }
// 形式二
if (condition) { code } 
else { code }
// 形式三
if (condition1) { code }
else if (condition2) { code }
else if (condition3) { code }
...
else { code }
```

这可能有点抽象，不好理解。我们一个一个看

### if形式

这是if判断语句的最简单的形式，含义为：如果条件判断为真，则执行之后的语句。下面看一个例子

```js
var x = 5,
    y = 7;
if (y > x) {
    console.log('y > x');
}
```

应该可以猜到会输出什么吧，没错，会输出`y > x`，这就是最简单的if形式

### if-else形式

else是否则的意思，所以，用if-else语句的话，必定有一条语句会执行，当然，理解也很好理解，就是如果条件判断为真，则执行if后的语句，否则执行else后的语句

```js
var x = 999;
var y = 777;
if (x < y) {
    console.log('x < y');
} else {
    console.log('x > y');
}
```

这样的话，if里面的`x < y`会判断为假，不会执行后面的语句，而会执行else后面的语句，输出`x > y`

### if - else if - else形式

这是if语句里面最复杂的形式了，不过虽然是最复杂的，但依然很好理解。如果第一个if里面的条件判断为真，则执行后面的代码，如果为假，继续判断第二个...以此类推，直到判断得到一个条件为真，或者全部为假，如果后面有else，则执行这个else里面的语句，如果没有，所有的代码都不执行

```js
var a = 0;
var b = 7;
var c = '9';
if (a > 0) {
    console.log('a > 0'); // 显然这句话不会执行
} else if (b < 7) {
    console.log('b < 7'); // 显然这句话也不会执行
} else if (typeof c === 'number') {
    console.log('c 是 number'); // 这句也不会
} else { // 这个else可以没有
    console.log('以上条件均为假'); // 这句会输出
}
```

这样，相信你已经对if语句有了基本的了解

### if语句的缩写形式

在js中，每写一个if都要接一个`{}`包裹的代码块显得很麻烦，所以if语句拥有其缩写形式

#### js语句与代码块

在此之前，我们需要先了解一下js中的语句和代码块

在js中，一行可以有一个或多个语句，如果一行有多个语句，语句间必须用`;`隔开，最后一个可以不加`;`，当然我们推荐所有语句都加上分号

```js
var x = 0; x++; x = 3; // 这是三个语句写在了一行
```

语句应该很好理解，就是计算机可以直接执行的一句话，比如赋值、运算、判断、调用等

下面是代码块

```js
{
    console.log('现在在代码块里面');
}
```

没错，由`{}`包裹的代码就称为代码块。这样，你现在应该了解代码块和语句是什么了，下面我们就可以看if语句的缩写了

#### if缩写

if后面通常会跟着代码块，但是，如果代码块里面只有一句话，就显得很别扭。所以，我们可以使用缩写

```js
if (true) console.log('这是if缩写语句里面输出的一句话');
```

缩写后的if语句会执行紧跟其后的 __一条__ 语句，之后的语句均不在if管辖范围内

有一种情况需要特别注意一下

```js
if (false); console.log('这句话不在if管辖范围内！！！')
```

对，没错，一般情况下if语句后不要直接加分号，否则就会出现上面这种情况

## 二 条件语句

在js中，有一个条件语句，叫switch语句

```js
switch (expression) {
    case condition1:
        code;
        break;
    case condition2:
        code;
        break;
    default:
        code;
        break;
}
```

下面我们来具体分析一下

### 基础用法

在上述示例中，`expression`是指一个表达式，输出一个值，这个值要在之后判断。然后程序向下进行，遇见第一个`case`，会判断第一个`case`后的`condition1`与`expression`是否相等（不是弱相等），相等，则执行里面的`code`，否则继续向下判断，如果所有的`case`都不成立，则会执行`default`里面的内容（default也可以没有，就不会执行），`default`一般写在最下面

示例：

```js
var x = 100;
switch (x) {
    case 1:
        console.log('x === 1');
        break;
    case 10:
        console.log('x === 10');
        break;
    case 100:
        console.log('x === 100');
        break;
    default:
        console.log('x不等于1 10 100');
        break; // 这个break可有可无，推荐加上
}
```

显然这个程序会输出`x === 100`。你可以自己尝试用上面的运行过程分析一下这个程序是怎么运行的

### 注意事项

这里我们要提一下这个`break`

这个`break`是用来阻止程序继续运行的，用于阻断`case`里面的语句的运行。如果没有这个`break`，程序的运行将会一直贯穿下去，直到遇见`break`，才会停止。我们可以类比一下来说明

`switch`可以看做是一个筛选某个东西（比如球）机器，圆括号是一个放入球的地方，括号里面的值便是我们放入的球，当放入时，球会向下落，直到遇到符合条件的`case`，这时球会进入中间，继续下落，直到遇见一个挡板——`break`才会停止，而中间球走过的地方，都会执行代码，如果没有`break`，球将会一直下落至机器底部

示例

```js
switch (99) {
    case 99:
        console.log('x === 99');
    case 88:
        console.log('x === 88');
    case 77:
        console.log('x === 77');
}
```

这样的话，程序会将这三个内容全部输出，因为没有`break`来阻止程序的继续进行

## 三 循环

在js中，共有三种基础的循环形式——`for` `while` `do while`

### while循环

`while`循环是一种常见的循环，格式如下

```js
while (condition) {
    code;
}
```

#### 基础使用

这是前置条件循环，意思是先判断`condition`是否为真，为真则执行后面的 __代码块__ 或 __一条__ 语句，执行过一次以后再次判断条件，以此类推，直到条件判断为假，终止循环

```js
var x = 0;
while (x < 100) {
    console.log(x);
    x++;
}
```

以上语句便会在控制台一次输出0 1 2 3 ... 98 99，很好理解吧 :）

#### 反复调用

你可以通过`while`循环反复执行某个语句，直到语句运算结果为`false`

```js
var x = 50;
while (x--);
```

和if缩写类似，在括号后面直接写一个分号可以不执行后面的语句，也就是说，这个循环会不断的执行`x--`，直到`x--`返回一个`false`，也就是`0`，才会终止运行

#### 死循环

当`while`中的条件始终为`true`且无法跳出时，会形成死循环局面

```js
while (true) console.log('1');
```

这样，程序会不断输出1而不会停止，也不会执行之后的语句，出现浏览器页面卡死的现象

### do while循环

`do while`循环是后置条件循环，格式如下

```js
do {
    code;
} while (condition);
```

这个循环并不是很常用，因为能用`do while循环`的程序，`while循环`都能做到

#### 基础使用

这是后置条件循环，会先执行`do`后面的代码块中的内容，执行完毕后，检测`condition`是否为真，为真则继续执行，否则终止运行

```js
var x = 1;
do x++; while (x < 10);
```

这是一个简写后的语句，当然也可以在`do`后面跟代码块，像格式里面一样

### for循环

`for`循环是一个极其常用的循环，也是一个非常重要的循环，格式如下

```js
for (statement1; condition; statement2) {
    code;
}
```

#### 基础使用

我们可以先看一个例子，通过例子来说明执行过程

```js
for (var i = 0; i < 10; i++) {
    console.log(i);
}
```

这个代码会依次输出0 1 2 ... 8 9，下面我们来进行分析

首先，for循环在执行时，会先执行圆括号里面的第一个语句`statement1`，这个语句一般是赋值语句，然后判定第二个语句`condition`，如果结果为真，则执行下面的代码，为假，则不再执行，执行完毕后，执行第三个语句`statement2`，一轮循环结束，下面会继续判断第二个语句`condition`，为真则继续执行，为假则不再执行，以此类推

那么，放到这个例子里面就是，首先执行了`var i = 0`，声明了变量i，然后进行判断`i < 10`，判断为真，执行`console.log`，输出0，代码块内代码执行完毕，执行`i++`，i就变成了1，第一轮循环结束。然后再次判断`i < 10`，为真，继续执行`console.log`，执行`i++`......直至`i < 10`判断为假

现在，你应该对`for循环`有了一定的了解了，下面我们看一些特别提醒

#### 省略

可以看到，`for循环`的圆括号里面有三个语句，其中所有的可以省略，我们看一些省略的情况

```js
// 省略第一个语句
var i = 0;
for (; i < 10; i++) console.log(i); // for循环也可以简写

// 省略第三个语句
for (var i = 0; i < 10;) console.log(i++);

// 两个都省略
var i = 0;
for (; i < 10;) console.log(i++);

// 省略中间 相当于while(true)
for (;;) console.log(1); // 这个循环是一个死循环
```

如果省略语句，那么直接写一个分号就行，注意：省略第三语句时后面不要有分号，for循环的圆括号里面必须只能有2个分号

### continue & break

这两个是循环里面的关键字，十分重要！！！

#### continue

这个关键字的意思是直接进行下一轮循环，意味着程序遇到这个关键字时，会无条件地终止本轮循环，直接开始下一轮循环

```js
for (var i = 0; i < 10; i++) {
    if (i === 5) continue;
    console.log(i);
}
```

以上例子中，当`i`等于5时，会执行`continue`语句，导致本轮循环立刻结束，不会执行下面的`console.log`，因此也不会输出5，最终输出的结果就是0 1 2 3 4 6 7 8 9

#### break

这个关键字的意思是直接跳出循环，意味着程序遇到这个关键字时，会无条件地终止循环的执行，执行循环之后的语句

```js
for (var i = 0; i < 10; i++) {
    if (i === 5) break;
    console.log(i);
}
```

上述例子中，当`i`等于5时，就会执行`break`语句，直接跳出循环，意味着之后的6 7 8 9都不会再输出，最终的结果就是0 1 2 3 4

## 四 注释代码

注释是代码中必不可少的内容，虽然程序不会执行它，但是它可以给读代码的人给予提示

### 注释方法

在js中，我们使用`//`来注释单行代码

```js
// 这就是一条注释
console.log(1); // 当然注释也可以放在这里
```

我们也可以用`/*  */`来注释多行或单行代码

```js
/*
这是一个多行注释
这里也在注释范围内
*/
console.log(/* 这是一个单行注释，不影响程序的正常运行 */1);
```

还有一种特殊的注释，这里我们不做深究，了解即可。这种注释叫做`js doc注释`，它的注释方法是`/**  */`，这种注释会被vscode等编辑器读取，来提供`自动补全`与`api说明`及`类型标注`等

```js
/** 
 * 这是一个js doc注释
 * @param {number} x 这种语句可将函数的参数x标注为number类型，了解即可，不必深究
 */
```

### 注释代码来阻止语句的运行

在有些情况下，我们需要让一个语句不执行，我们不必删掉它，况且删掉以后需要的时候还需要重新写，我们只需要用`//`或`/* */`将语句注释掉，就可以让程序不再运行了
